This module focuses on

1. introduction to the programming of POSIX Inter-process Communication (IPC) on Unix.
2. concept description of Signals, Pipes and Message Passing.
3. use examples of these concepts in Unix/Linux programming
4. examine several classical process-synchronization problems such as deadlock
5. introduction to advanced IPC methods such as message queues.  

This forms the core component to the subject.

At the end of this module, you should be able to answer these questions under Unix and POSIX:

1. What is the Signal, how to create, send and receive the Signal?
2. What are the blocked signals and their functions such as sigprocmask, sigsuspend and kill?
3. How does the Pipe and FIFO work?
4. What is the difference between the Pipe and FIFO?
5. What is the message queue?
