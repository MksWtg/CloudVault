Functional review checklist
1. Creating a PR against main/master triggers the Build and Test workflow automatically
 2. Manually triggering Build and Test via workflow_dispatch works
 3. Build succeeds and the exe is stamped with a GitVersion-derived version (not 0.0.0)
 4. Tests run and pass
 5. SQL Server is detected/started and SQL_SERVER_INSTANCE is correctly passed to the tests
 6. Test results .trx are uploaded as an artifact
 7. Code coverage HTML report is generated and uploaded as an artifact
 8. test-report job runs even if tests fail (if: always())

Patch Release Workflow
- some info: the `releases/release-X.Y` branch is created when a major or minor release workflow is run. if a bug needs fixing, you can cherry pick commits from main onto that branch and then run patch release from that branch to publish 1.2.1, 1.2.2
9. Running patch release from a releases/release-X.Y branch succeeds
10. Running patch release from any other branch fails with a clear error message
11. The MSI is signed
12. The exe is signed
13. The Chocolatey .nupkg is created with the correct version
14. The .nupkg is pushed to CHOCOLATEY_FEED_URL 15. A GitHub
release is created with the MSI and nupkg attached

Minor Release Workflow 16. Running minor release from main succeeds 17. Running minor release from a non-main branch
fails with a clear error 18. A releases/release-X.Y branch is created and pushed 19. Version increments the minor
number correctly (e.g. 1.0.0 → 1.1.0) 20. GitHub release created targeting main

Major Release Workflow 21. Running major release from main succeeds 22. Version increments the major number
correctly (e.g. 1.2.3 → 2.0.0) 23. Release branch named releases/release-X.0 (e.g. releases/release-2.0) is created

Packaging 24. Building locally without passing AssemblyVersion doesn't error (wixproj falls back to 0.0.0.0) 25. MSI
installs cleanly on a fresh Windows machine 26. MSI upgrades a previous installation (old version is removed) 27.
Installed exe is on PATH and callable

GitVersion 28. On a WI branch, version has a pre-release label (e.g. 0.0.1-WI01234.1+...) 29. On main, version has
no pre-release label



The paradox of low level skills

- adults are generally better at higher level thinking and logic, this allows us to be more present when developing skills- be conscious about mistakes and so on. more life experience so easier to understand consciously abstract concepts. especially formal stuff, like advanced math, not intangible stuff like pattern recognition.
- This makes us think we should be better at learning when we are adults
- But kids not only have an abundance of time, they have no respect for their time -> a kid can spend 100s of hours shamelessly grinding out something just because they like it
	- coding
	- CAD
	- guitar chords
	- grinding out low level mathematics computations such as solving differential equations, integrals, matrix multiplications
